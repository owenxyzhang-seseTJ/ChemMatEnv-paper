🧠 核心设计理念

🏗️ 故事先行

正式起草前，先锁定 8 个字段的 story package：核心问题、瓶颈、设计原则、主 claim、证据映射、机制定位、范围边界、目标期刊画像。避免边写边改故事，导致论文"看起来流畅但逻辑失控"。

🧱 结构优先于措辞

采用 失败模式诊断（9 项预检清单），按 paper type → section job → paragraph logic → claim/evidence → sentence polish 层级修复。结构错了就绝不 polish 句子。

🔍 实时检索，不凭记忆

期刊要求（word limits, template, display items）和同主题顶刊表达（collocations, hedge patterns, discourse moves）必须在起草前实时搜索，找不到就标 not found，不猜测。

⚖️ 闸门分级，避免循环

Full gate（含 live 文献搜索）：新起草 / 大改后 / 最终轮。Light gate（仅 anti-AI + logic）：小修措辞。同 session 同主题搜索结果可复用，不必每段重搜。

📋 9 步操作流程详解

1
Orient — 识别稿件阶段
判断当前任务属于 audit_only / story_architecture / journal_requirements / manuscript_drafting / section_revision / full_manuscript_revision / review_cycle / response_revision 之一，声明后进入对应流程。
2
Read References — 按需加载参考文件
14 个 reference 文件按任务加载，不全量加载。故事/claim 控制、章节 playbook、citation search、表达审计、去 AI、逻辑修复、失败模式诊断、LaTeX 排版、审稿协议、矩阵 schema —— 只加载当前任务需要的。
3
Build Or Rebuild The Story — 锁定 story package
提取证据 → 2-3 个候选故事 → 选默认故事并解释弱故事为何弱 → 设计标题/摘要/图序/贡献尺度/危险逻辑跳跃 → 锁定 8 字段。见 references/story-architecture-protocol.md。
4
Research Target Journal — 实时期刊要求检索
搜索 official author guidelines → 记录 word limits / abstract 格式 / display items / template / required sections / SI 规则 → 按 ACS/RSC/Wiley/Nature/Science/Cell Press 六大家族分别适配。见 references/target-journal-requirements.md（含 6 家族速查表）。
4b
Citation Search — 文献引用搜索（可选）
分段文本 → 解析 claim → 搜索（精确/同义/背景三档）→ 5 级保守评估 → 导出 ENW/RIS/RDF → HTML 可视化报告。期刊范围覆盖材料/化学/环境六大出版家族。仅用户要求或 claim 缺引用时触发。见 references/citation-search.md。
5
Draft Or Revise — 按 Playbook 起草
预检失败模式 9 项 → 按 references/per-section-playbooks.md 的章节结构（Abstract 六步 / Introduction 四段漏斗 / Results 证据阶梯 / Discussion 解读+hedging / Conclusion 三部分收尾 / Title 三种期刊风格）和 paper-type argument chain 起草。中文草稿先跑中→英修复模式（8 种常见模式）。
6
Run Quality Gates — 质量闸门
Full gate（新起草/大改/最终轮）：顶刊表达检索+diction校准 + 去AI味 + 逻辑链审计+反向提纲。
Light gate（小修）：仅去AI味 + 逻辑链。输出 quality_gate_report。闸门通过 ≠ checklist 打勾 —— 必须真正修改了正文才算通过。
7
Handling Editor Pass — 编辑预审
模拟期刊处理编辑：scope fit / desk-reject 风险 / novelty / evidence gaps / overclaim / 是否值得送审 → 生成 review_action_matrix → 立刻修订。
8
Three Reviewer Rounds — 三轮三审稿人
每轮：A（方法和证据）+ B（领域和新颖性）+ C（逻辑和风格）独立评审 → 合并 → 修订 → 重跑质量闸门 → 遗留项带入下轮。reviewer 输出格式见 references/multi-round-review-protocol.md。不偷偷让 reviewer 改写故事。
9
Final Output + LaTeX Layout — 最终输出
修订后稿件 + locked story package + journal requirements + editor verdict + 三轮 reviewer 历史 + review_action_matrix + quality_gate_report + risk register。可选 LaTeX 排版诊断（float / 宽扁图 / 搁浅标题 / 多面板 / SI 密度）。靠写作无法解决的证据缺口直接标注，不用语言包装。

📑 五种论文类型 Argument Chains

🔬 Research Paper

field need → target property gap → design rationale → synthesis → characterization → property vs. benchmark → mechanism → implication → boundary

材料/化学最常用类型。起草顺序：Results → Intro/Conclusion → Title → Abstract。

🧪 Hypothesis / Mechanism

phenomenon → candidate explanations → explicit hypothesis → falsification path → evidence for/against → rival explanations → bounded conclusion

机制论文核心：必须明确声明假说，区分支持性 vs 一致但非鉴别性证据。

🛠️ Methods / Technique

task/problem → limits of existing → proposed method → evaluation (reliability / speed / resolution) → reproducibility → boundary

禁止 vague 短语：under standard conditions / using routine methods / the method was validated。

📚 Review

scope & criteria → organizing principle → synthesis by argument → disagreements & gaps → author's stance → outlook with open questions

替换 "Author A reported X. Author B reported Y." 为按 mechanism/method/conclusion 分组合成。

💻 Computational / Modeling

task/scope → what the model is → why it works → evaluation vs. experimental ref → component analysis → failure modes → applicability boundary

DFT/MD: report functional, basis set, convergence, comparison to experiment。ML: report training set, test performance, extrapolation limits。
