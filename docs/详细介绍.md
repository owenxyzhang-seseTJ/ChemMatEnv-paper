# ChemMatEnv-paper 详细介绍

## 一、背景与定位

ChemMatEnv-paper 是一个为材料科学、化学、环境科学与工程三大领域量身定制的顶刊论文全流程 AI 协作技能（Skill）。它的前身 `top-journal-review-cycle` 已具备故事包设计、实时期刊要求检索、顶刊表达校准、去 AI 味、逻辑链审计和模拟审稿循环等核心能力。v0.2.0 深度融合了 nature-skills 系列中的 nature-writing、nature-polishing、nature-citation 三大工作流，新增了文献引用搜索、LaTeX 排版诊断、失败模式诊断、章节 Playbook、中→英修复模式、反向提纲、六大期刊家族速查表、Per-Journal Diction 校准等模块。

本技能遵循 **Agent Skills 开放标准**（YAML frontmatter + Markdown body），可在 ZCode、Claude Code、Codex CLI、OpenCode、Hermes、Gemini CLI、Cursor、Windsurf、Cline、GitHub Copilot 等 20+ 工具中直接使用。三套插件清单（`.zcode-plugin/`、`.claude-plugin/`、`.codex-plugin/`）确保与主流平台的自动发现机制兼容。

## 二、核心理念

1. **故事先行，锁定后不变。** 正式起草前，先用 8 字段 story package 锁定论文的核心命题。审稿过程中严禁 reviewer "悄悄改写故事"——如果故事需要变化，必须显式重建，而不是在修改措辞中渐进漂移。

2. **结构优先于措辞。** 采用 9 项失败模式预检清单，按 `paper type → section job → paragraph logic → claim/evidence/boundary → sentence polish` 层级修复。章节职责错了就绝不润色句子。

3. **实时搜索，不凭记忆。** 期刊要求（word limits、template、display items）和同主题顶刊表达（collocations、hedge patterns、discourse moves）必须在起草前实时搜索。找不到就标记 `not found`，不猜测。

4. **闸门分级，避免循环。** Full gate（含实时文献搜索）仅在新起草、结构性大改、最终轮时运行。Light gate（仅反 AI + 逻辑链）用于小修。同一 session 同主题的搜索结果可复用，不必每段重搜。

## 三、架构概览

整个技能由 1 个核心 SKILL.md（约 230 行）和 14 个 reference 文件组成，按需加载：

### 故事与声明控制层（3 个文件）

- **`story-package-schema.md`**：8 字段定义 + 4 条 Red Flags（标题声称范围超实测数据、机制声明缺独立证据、性能→设计规则无桥接、仅与弱基线比较声称领域级推进）。
- **`story-architecture-protocol.md`**：8 步故事设计流程——提取 3-5 个核心观察→上证据阶梯→识别 field-level problem→收窄到 bottleneck→定义 design principle→提 2-3 个候选故事→选默认+解释弱故事→设计标题/摘要/图序/期刊定位。
- **`claim-evidence-ladder.md`**：5 级证据阶梯（Observation → Comparative → Structural → Mechanistic → Scope），含晋升规则和化学/材料特有关系词校准。

### 起草层（3 个文件）

- **`per-section-playbooks.md`**：6 个章节的详细 playbook（Abstract / Introduction / Results / Discussion / Conclusion / Title），5 种论文类型 argument chains（Research / Hypothesis / Methods / Review / Computational），全部针对材料化学适配。
- **`manuscript-drafting-protocol.md`**：起草规则 + 指向 per-section-playbooks。
- **`target-journal-requirements.md`**：要求检索协议 + 6 大期刊家族速查表（ACS/RSC/Wiley/Nature/Science/Cell Press），含 article types、关键约束和常见 desk-rejection 触发模式。

### 质量闸门层（4 个文件）

- **`live-literature-expression-audit.md`**：来源优先级 4 级 + 搜索词组合 + 5 类语言学要素 + Per-Journal Diction 校准方法。
- **`anti-ai-style-rules.md`**：6 类 Blocked Patterns + 12 词删除/替换清单 + 5 条重写规则 + 8 种中→英修复模式 + 3 种材料化学专用修复。
- **`logic-link-audit.md`**：句子级 8 种关系 + 段落级 7 种关系 + 反向提纲（5 问读者测试 + 5 步工作流 + 5 种修复动作）。核心改进：不逐句贴标签，只在 `unsupported_jump/unsupported_shift` 处标注。
- **`failure-mode-diagnosis.md`**：编辑前 9 项必检清单 + 优先级层次 + 5 种材料化学特有问题。

### 审稿层（2 个文件）

- **`multi-round-review-protocol.md`**：Reviewer 输出格式（推荐意见四档 + top 3-6 issues + 是否需新数据）+ 停止条件 + Subagent 数据传递细则。
- **`review-action-matrix-schema.md`**：8 列 matrix schema + 取值枚举 + 4 条治理规则。

### 扩展层（2 个文件）

- **`citation-search.md`**：7 步引用工作流 + 6 家族期刊白名单 + 5 级保守评估 + 中文模式 + 材料化学特有失败模式清单。
- **`latex-layout.md`**：诊断工作流（编译→log 三信号→contact sheet→量 aspect ratio）+ 6 种修复模式 + 材料化学领域特有问题（XRD 宽扁 4:1、TEM grid 2×2 布局、SI 密度）。

## 四、操作流程

完整的 9+1 步操作流程：

**Step 1 — 识别稿件阶段。** 判断当前任务属于 audit_only / story_architecture / journal_requirements / manuscript_drafting / section_revision / full_manuscript_revision / review_cycle / response_revision 之一，声明后进入对应流程。

**Step 2 — 按需加载 Reference。** 14 个文件按任务选择性加载，不全量加载。

**Step 3 — 构建或重建故事。** 提取证据→2-3 个候选故事→选默认并解释为何不选弱故事→锁定 8 字段 story package。故事变了就停止写作重建，不要边写边改。

**Step 4 — 目标期刊要求检索。** 搜索 official author guidelines→记录 limits/template/required sections→按六大家族分别适配。

**Step 4b — 文献引用搜索（可选）。** 仅在用户要求或发现 claim 缺引用时触发。分段→解析→搜索→5 级评估→导出→HTML 报告。

**Step 5 — 按 Playbook 起草。** 预检 9 项失败模式→按章节 playbook 和 paper-type argument chain 起草。中文草稿先跑中→英修复模式。

**Step 6 — 运行质量闸门。** Full/Light 分级执行。输出 `quality_gate_report`。闸门通过≠checklist 打勾——必须真正修改了正文才算。

**Step 7 — Handling Editor 预审。** scope fit / desk-reject 风险 / novelty / evidence gaps / overclaim / 是否值得送审。

**Step 8 — 三轮三审稿人模拟。** A（方法和证据）+ B（领域和新颖性）+ C（逻辑和风格），独立评审→合并 matrix→修订→重跑质量闸门→遗留项带入下轮。

**Step 9 — 最终输出 + LaTeX 排版诊断。** 修订稿件 + locked story package + journal requirements + editor verdict + 三轮 reviewer 历史 + matrix + quality_gate_report + risk register。可选 LaTeX 排版诊断。

## 五、跨平台兼容

三套插件清单覆盖主流发现机制：

| 平台 | 发现方式 |
|---|---|
| ZCode | 读 `.zcode-plugin/` → `.claude-plugin/` → `.codex-plugin/` |
| Claude Code | 读 `.claude-plugin/plugin.json` |
| Codex CLI | 读 `.codex-plugin/plugin.json` |
| OpenCode / Hermes / Cursor / Windsurf / Cline / Copilot | 通过 `~/.agents/skills/` 共享目录 |

同一套 SKILL.md + references/ 即插即用，无需平台适配。

## 六、安装与使用

```bash
git clone https://github.com/owenxyzhang-seseTJ/ChemMatEnv-paper.git
# ZCode / Claude Code / Codex CLI：添加整个目录为本地插件
# 其他工具：
mkdir -p ~/.agents/skills
ln -s $(pwd)/ChemMatEnv-paper/skills/ChemMatEnv-paper ~/.agents/skills/
```

触发方式：在任意兼容平台的对话中输入 `ChemMatEnv-paper` 或描述投稿相关任务即可自动激活。

**典型用法：**

- 「帮我设计这个故事线」
- 「检索 JACS 的作者指南」
- 「去 AI 味 + 修逻辑」
- 「帮这段 Abstract 校准 JACS 的表达」
- 「找引用支持这个声明」
- 「看看 LaTeX 排版有什么问题」
- 「模拟一轮审稿」
